Je veux créer le module front d'une application dont le nom est "Magrit" et qui sera composée de 2 modules que je vais décrire ci-dessous, et dont l'objectif est d'abord de fournir un prix pour n'importe quel produit qui s'imprime ou se fabrique (imprimés, livres, textile, packaging, grand format, impression numérique etc.). 
La réponse doit fournir les informations techniques telles que les gabarits des fichiers, les visuels ou les modèles 3D pour agrémenter les informations associées aux prix pour l'ensemble des produits demandés par les utilisateurs. 
Ainsi le service peut fournir toutes les fonctionnalités requises pour acheter et vendre la production print. Les commerciaux d'imprimerie qui ainsi peuvent réaliser facilement et rapidement touts les chiffrages requis pour leurs clients, 
De même que les sites de ventes en ligne qui pourront intégrer le module "prompt" de la solution pour remplacer le "search" traditionnel des CMS e-commerce par un prompt qui verra le service renvoyer les informations produits et les prix. Le service peut aussi s'adresser aux acheteurs B2B qui ont besoin de prix pour des quantités importantes de produits. 

Les 2 modules de la solution sont : 
1-Le module front que nous allons concevoir ensemble, sur le principe d'une interface de prompt IA comme Claude par exemple, permettra à l'utilisateur de faire des requêtes qui vont concerner le domaine du web-to-print. 
2-Le deuxième module dit "clariprint" sera ajouté par la suite, c'est un moteur de calcul spécialisé dans le domaine de l’imprimerie et une solution dite ( site web : https://www.expert-solutions.fr/clariprint/) Configure Price Quote.

Concernant le module du site web front, je veux que sur la base du prompt initial de l'utilisateur, il lui soit présenté des résultats, les plus professionnels par rapport à sa demande. Il faut que les résultats soient présentés selon la logique d’une fiche produit sur un site e-commerce de print comme https://mixam.co.uk ou https://www.vistaprint.fr/supports-marketing/chemises-porte-documents .
Si il y a plusieurs produits alors il faut les présenter comme dans un panier d’un site e-commerce en pouvant éditer chacun des onglets qui pourront ainsi permettre de modifier les options du produit.

Ainsi si il demande : 500 cartes de visites, il faut dans un premier temps interpréter une réponse sur la base de ce qui est le plus vendu dans ce domaine, et structurer les informations produits selon le principe décrit ci-dessus.

Ce pour lui proposer en même temps que la réponse, des suggestions pour améliorer son produit. Par exemple ici, un pelliculage brillant, des coins ronds, etc. 
Il faut que les réponses soient fournies sur la base d'un tableau avec 4 onglets pour :
1-Fiche produit 
2-Afficher le prix
3-Mokup et 3D  
4-Éditer le formulaire 

Je charge 3 images pour chacun de ces onglets, sachant que le bouton afficher le prix doit l'indiquer en haut à droite de la page dans un bloc de type "panier" d'achat d'un site e-commerce.
 
Dans un premier temps tu vas mettre des données que tu trouveras sur le web, dans les onglets sachant que ce sera le rôle du module Clariprint de fournir :
-Le prix 
-Les gabarits de documents 
-Les mockups et 3D
-Les données à afficher dans le formulaire 

sur la base du contenu structuré que tu lui enverras pour décrire les produits choisis par l'utilisateur et qu'il devra calculer. Au terme de son calcul il enverra les données à afficher dans les onglets.
