04

Règles d'usage

Ce qu'on fait · ce qu'on ne fait pas
On fait
  • Display et titres en weight 100-300, jamais au-dessus.
  • Body long en weight 300 avec line-height 1.6.
  • Letter-spacing négatif sur tout ce qui dépasse 24px (-0.02 à -0.04em).
  • Le 500/700 réservé aux prix, totaux, statuts critiques.
  • JetBrains Mono pour les métadonnées techniques (IDs, specs, timestamps).
  • Antialiasing activé (-webkit-font-smoothing: antialiased).
On évite
  • Display en 500 ou 700 — trop "press release".
  • Body en 400 si le texte dépasse 3 lignes — alourdit.
  • Italic véritable : on utilise la graisse 300 comme voix douce.
  • Tracking positif sur du body — Helvetica Neue est déjà ouverte.
  • Mélanger avec une autre famille display (Fraunces, Newsreader, etc.).
  • Graisses 600+ sur les interfaces denses (admin, table de devis).

Tokens CSS

Prêt à coller 
:root {
  --mg-font-sans: "Helvetica Neue", "HelveticaNeue",
                  "Inter Tight", Helvetica, Arial, sans-serif;
  --mg-font-mono: "JetBrains Mono", ui-monospace, monospace;

  /* Weights */
  --mg-w-thin:       100;   /* display XL uniquement    */
  --mg-w-ultralight: 200;   /* titres de page           */
  --mg-w-light:      300;   /* body long · sous-titres  */
  --mg-w-regular:    400;   /* body court · UI labels   */
  --mg-w-medium:     500;   /* prix · CTA · accents     */
  --mg-w-bold:       700;   /* alertes · rare           */

  /* Scale */
  --mg-fs-display: 88px;  --mg-lh-display: 0.98;
  --mg-fs-h1:      56px;  --mg-lh-h1:      1.02;
  --mg-fs-h2:      36px;  --mg-lh-h2:      1.10;
  --mg-fs-h3:      22px;  --mg-lh-h3:      1.25;
  --mg-fs-body:    16px;  --mg-lh-body:    1.60;
  --mg-fs-meta:    13px;  --mg-lh-meta:    1.50;
  --mg-fs-cap:     11px;  --mg-lh-cap:     1.40;

  /* Letter-spacing */
  --mg-ls-display: -0.04em;
  --mg-ls-h1:      -0.035em;
  --mg-ls-h2:      -0.025em;
  --mg-ls-h3:      -0.015em;
  --mg-ls-cap:      0.08em;
}
