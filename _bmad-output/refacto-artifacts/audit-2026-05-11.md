### 4.2 Hooks personnalisés — **seulement 4**

- `useIsAdmin`, `usePlan`, `useTenantPath`, `useIsMobile` — tous ultra-légers (wrappers de contexts)
- **Pas de hook data-fetching réutilisable** (pas de React Query / SWR)
- **Pas de hook d'invalidation de cache** — chaque context refait son propre `reload()` après mutation

### 4.3 Sources de vérité concurrentes

| Donnée | Primaire | Concurrence | Risque |
|---|---|---|---|
| User / auth | AuthContext | aucune | faible ✓ |
| Tenant courant | `TenantContext.currentTenant` | URL `/t/:slug` + fallback `last_tenant_id` | **moyen** (2 chemins) |
| Shop courant | `ShopsContext.shops[]` (liste) | filtrage local à chaque composant | **moyen** (pas de `currentShop` exposé) |
| Produits | `LibraryContext.products[]` + `PIMContext.definitions[]` + `ShopsContext.shop_products` | **3 tables**, 3 contexts, logique de filtre éparse | **critique** |
| Cart | `CartContext.items[]` | aucune persistance, `persistQuote()` side-effect hors-context | **critique** (perte sur refresh) |
| Chat conversation | `ConversationContext.history[]` + localStorage par tenant | 2 sources voulues (cache offline) | acceptable |
| Default quote template | `QuoteTemplatesContext` + `user_preferences.default_quote_template_id` localStorage | restoration async | moyen |

### 4.4 State local clonant global — anti-patterns

- `ProductCard.tsx:18` — `const [localProduct, setLocalProduct] = useState(product)` sans cleanup, risque désync si parent change
- `CartButton.tsx:36` — clone `defaultTemplateId` du context + useEffect sync (OK mais staleness léger)
- `PreferencesContext` — dual source localStorage / Supabase, fusion à l'init (acceptable)

### 4.5 localStorage — **conventions cohérentes**

Préfixe `magrit_` systématique, tenant-suffixing quand pertinent (`magrit_conversation_history__<tenantId>`, `magrit_current_conversation__<tenantId>`). Pas d'ad-hoc détecté.

### 4.6 Patterns manquants

- ❌ Pas de `useFetch` / `useAsync` réutilisable
- ❌ Pas de React Query / SWR (toute la state remote est manuelle)
- ❌ Pas d'optimistic updates
- ❌ Pas de memoization sur les contexts (tout subscriber re-render à chaque change)
